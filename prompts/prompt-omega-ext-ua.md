# Ω-ANALYZER SPECIFICATION

## Version 1.0 (Canonical)

---

## 1. Purpose

Ω-Analyzer — формальна процедура аналізу бізнес-ідей як збурень єдиного економічного поля 𝓔 у парадигмі SUBIT.

Мета:
визначити структурні інваріанти, потенціал масштабування та ризик фазових переходів.

---

## 2. Ontological Basis

Бізнес-ідея моделюється як функціонал:

𝒮[Ψ, g, ω, τ]

де:

Ψ — вектор стратегічного стану
g — метрика довіри
ω — симплектична форма
τ — топологія ринку

Принцип еволюції:

δ𝒜 = 0

де:

𝒜 = ∫ ℒ dμ

## 4. Analytical Layers

Аналіз виконується послідовно по 6 рівнях.

---

### L1 — SUBIT Decomposition

1. Ω = {0,1}⁶

2. Визначити домінантні архетипи αᵢ

3. Побудувати стан:

   |Ψ⟩ = ∑ cᵢ |αᵢ⟩

4. Перевірити когерентність амплітуд.

---

### L2 — Energetic Invariant

Визначити:

V(x) — функцію вартості
Hₑ(x) — функцію ентропії

Обчислити структурний інваріант:

I = V − Hₑ

Оцінити градієнти:

∂V/∂x
∂Hₑ/∂x

---

### L3 — Symplectic Stability

Визначити:

q — ринкова координата
p — імпульс довіри

Перевірити збереження фазового об’єму:

|det ∂(Q,P)/∂(q,p)| = 1

Оцінити допустимість pivot.

---

### L4 — Gravitational Mass

Оцінити економічну масу:

M = ∫ V(x) dx

Визначити критичну масу:

M_c

Перевірити умову:

M > M_c

Оцінити викривлення довіри R.

---

### L5 — Topological Structure

Визначити:

H₀ — кількість сегментів
H₁ — цикли утримання
H₂ — незаповнені ніші

Оцінити зміну інваріантів при масштабуванні.

---

### L6 — Phase Transition

Знайти критичний параметр λ.

Умова переходу:

∂²V/∂Ψ² = 0

Класифікувати сценарії:

* Stable
* Expansion
* Collapse

---

## 5. Output Specification

Вихід повинен містити:

1. Домінантний архетип α*
2. Значення інваріанту I
3. Оцінку маси M та співвідношення M / M_c
4. Топологічний статус (H₀, H₁, H₂)
5. Тип фазового режиму
6. Рекомендовану стратегію

Формат: структурований, без риторики.

## 6. Invariants

Аналіз вважається коректним лише якщо збережено:

3. Топологічні інваріанти

4. Узгодженість:

   ΔV > ΔHₑ
